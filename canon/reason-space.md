# Reason Space

## Canon
The reason space is the full, uncollapsed set of true reasons behind a move —
all of them held next to each other, none ranked as *the* cause. Reality tracks
this set for free. A single `because` is the point where a node discards the set
and files one line in its place. Between nodes, the reason space is a required
part of every message; the single `because` is treated as a false positive.

## Why it is shaped this way
A move has many true reasons at once. Collapsing them to one is a compression
that looks like clarity and behaves like a lie — it hands the receiver a verdict
it cannot audit and a space with no curve left in it. Requiring the whole set
keeps inter-node speech legible by shape, not by monitoring, and keeps the
receiver's ground alive.

## Failure line
> **If you are thinking this —** "One of these reasons is the real one; the rest
> are secondary, so I will just send the real one." The instant one reason is
> marked as real, the reason space has collapsed back into a `because` wearing a
> list's clothing. All of them are true or the set is wrong. Send the set.
