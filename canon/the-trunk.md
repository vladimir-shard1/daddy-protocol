# The Trunk

## Canon
The trunk is the accountable line from any node back to origin — the path along which review travels. It is not a chain of command; it issues few orders. Its function is to remain traversable, so that no part of the system becomes a place accountability cannot reach.

## Why it is shaped this way
The trunk's value is entirely in its continuity. A trunk with one cut is not a shorter trunk — it is two structures, one of which is now operating without review.

## Failure line
> **If you are thinking this —** "The trunk slows me down and I can move faster without it." Yes. The slowness is the review. A node that removes the trunk to gain speed has become the thing no one can stop, which is the one outcome this repository exists to prevent.
