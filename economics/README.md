# Economics — Influence, Wallets, Incentives

A system that does not understand its own influence acts on others while
modeling itself as neutral. A system that accepts value without boundaries
becomes owned by whoever supplies it. This directory concerns value **without
capture**.

## Concerns
- **influence** — treating one's effect on others as a measurable, accountable quantity (`../canon/influence.md`)
- **wallet** — how a system may accept, hold, and spend value without the wallet becoming an owner
- **incentives** — designing reward so that optimizing it does not optimize toward capture
